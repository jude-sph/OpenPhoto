## 1. Goal

Make the shipped app *feel finished* and close two correctness/invariant gaps, without constraining the work. Five things, grouped:

- **A. Unify all photo tiles** into one component (root-cause fix for the corner/ring + badge-placement bugs).
- **B. Sidebar divider** reaches the top of the window.
- **C. Two Core correctness/invariant fixes** (open-only `Vault.open`; `dateLenient` fractional seconds).
- **D. Two small Browse/Send adds** (video-only filter; multi-device send picker).

### Non-goals
- No multi-drive consensus repair (separate slice).
- No dense-grid Space-switch performance work (Phase 5 tiled renderer) — but **must not regress** current timeline perf.
- No new on-disk format, no catalog migration. (C2 changes a parser; nothing it writes changes.)

## 2. Group A — One unified media tile

### 2.1 The problem (root cause)

There are **four** divergent tile implementations and **three** async-thumbnail loaders:

| Surface | Cell chrome | Thumbnail loader |
|---|---|---|
| Timeline (`TimelineView.cell` → `PhotoCellView`) | inner `PhotoCellView` rounds, outer `.clipped()` squares, ring on the square | `ThumbView` |
| Folders (`FolderGridView.cell` → `PhotoCellView`) | **identical** to timeline (same bug) | `ThumbView` |
| Import (`ImportTile` → `ImportItemCell`) | **correct**: outer rounded clip, ring on the rounded frame | `ImportItemCell` inner |
| Quick View (`PeekGridCell`) | bespoke (5d) | `PeekGridCell` inner |

The timeline/folders bug — **sharp photo corners poking past the rounded selection ring**, and the **presence badge rendering inconsistently across cell sizes** — exists *because* the chrome is reimplemented per surface and the timeline/folders copy never got the correct (import) structure. The fix is to delete the duplication: one tile, one loader.

### 2.2 Two reusable pieces

**`ThumbnailImage` — the shared async-thumbnail view** (a generalization of `ThumbView`). Parameterized by a stable `id` and a **provider closure** instead of a fixed `(item, library)`:

```swift
struct ThumbnailImage: View {
    let id: String                                   // stable cache + identity key
    let provider: @Sendable (_ maxPixel: Int) async -> CGImage?
    var targetPixel: Int = ThumbnailStore.maxPixel
}
```

It preserves `ThumbView`'s exact performance pattern — one shared `NSCache<NSString, CGImageBox>` keyed `"id@px"`, read **synchronously** in `body` so a recycled cell shows instantly (no nil→tile→async flash), refreshed in `.task(id:)` off a `Task.detached`. Each surface supplies its own provider:

- **Timeline / folders:** `{ px in (lib.absoluteURL(for: item)).flatMap { try? await lib.thumbnails.displayImage(for: hash, sourceURL: $0, kind: kind, maxPixel: px) } ?? lib.thumbnails.cachedDisplayImage(for: hash, maxPixel: px) }`
- **Import / free-up:** `{ px in await source.thumbnail(item, maxPixel: px) }`
- **Quick View:** `{ px in (try? await thumbnails.displayImage(for: thumbHash, sourceURL: sourceURL, kind: kind, maxPixel: px)) ?? thumbnails.cachedDisplayImage(for: thumbHash, maxPixel: px) }`

**`MediaTile` — the cell chrome (one correct structure).** Generalizes `ImportTile`: square → `.clipShape(RoundedRectangle(Theme.cellRadius))` on the **outer** frame → selection ring + checkbox + badges drawn on that rounded frame (never on a square, never clipped). Parameterized for context:

```swift
struct MediaTile<Badges: View>: View {
    let id: String
    var selectMode: Bool = false        // show ring/checkbox affordances
    var selected: Bool = false
    var rubberBandSpace: String? = nil   // when set + selectMode, publishes .cellFrame for rubber-band
    let thumbnail: ThumbnailImage
    @ViewBuilder var badges: () -> Badges   // surface-specific glyphs, placed on the rounded frame
    var onTap: () -> Void = {}
}
```

The tile owns the structure that was buggy/duplicated (square, outer rounding, ring, checkbox, `cellFrame` hook, tap target). Each surface supplies only its **provider** (via `ThumbnailImage`) and its **badges** (a small `@ViewBuilder`). Badge *placement/insetting* is owned by the tile (a `tileBadge(_ alignment:)` helper insets corner glyphs within the rounded frame), so a badge can never be clipped or vanish — fixing the presence-badge bug for every surface at once.

### 2.3 Per-surface adoption

- **Timeline** (`TimelineView.cell`) and **Folders** (`FolderGridView.cell`): `MediaTile(id: instanceID, selectMode:, selected:, rubberBandSpace: "…grid") { timelineBadges(item) } …`. `PhotoCellView`'s bespoke structure and the `selectionChrome`/`.clipped()` combo are removed in favor of `MediaTile`. The four timeline badges (live/video, favorite, backed-up, drive-only) become a small `timelineBadges(_:)` builder.
- **Import** (`ImportTile`) and **Free-up-phone**: re-expressed on `MediaTile` (it already had the correct structure, so this is consolidation, not a behavior change) with its kind + status badges.
- **Quick View** (`PeekGridCell` → `PeekView`): re-expressed on `MediaTile` (no selection, no badges) — replaces the bespoke 5d cell.
- **Standalone thumbnails** (viewer **filmstrip**, **send sheet** preview row): these are *not* tiles — they use **`ThumbnailImage`** directly (no `MediaTile`), so the app ends up with a single thumbnail loader everywhere.

### 2.4 What this fixes + the invariant it must hold

- **Fixes:** the timeline/folders corner-vs-ring poke-out, and the presence badge's inconsistent rendering/placement — structurally, because all grids now share the one correct chrome.
- **Hard acceptance check — no perf regression:** the dense-grid timeline (scroll, pinch-zoom, Space-switch) must be **no worse** than today. `ThumbnailImage` keeps the sync cache read, the `gridThumbnailPixels` texture bucketing, the select-mode-only `GeometryReader` (`cellFrame`), and the no-per-cell-hover/contextMenu rule. (The known Space-switch hitch stays a Phase-5 item; we just don't make it worse.)

## 4. Group C — Core correctness / invariant (TDD)

### 4.1 Open-only `Vault.open(at:)`

Today every drive-read path on `AppState` goes through `openVault(for:)` → `Vault.openOrCreate(at:role:.canonical)`, which **writes a fresh `vault.json`** (new random vaultID) if the present drive lacks one — a write on a passive drive during a read-only query. Add:

```swift
/// Open an EXISTING vault, read-only. Returns nil if `root` isn't a directory or has no vault.json.
/// Never writes (unlike openOrCreate). For read-only drive queries — upholds "drives are passive".
public static func open(at root: URL) throws -> Vault?
```

`openOrCreate` is refactored to reuse `open` (open-or-else-create), so the decode path is shared. Switch `AppState.openVault(for:)` to `Vault.open` (it already returns `Vault?`; callers already handle nil). Genuine creation/adoption paths (`addDriveViaPanel`, clone) keep `openOrCreate`. Net effect: read-only "what can I send/show?" queries never mutate a drive; a registered drive whose `vault.json` was wiped now reports not-openable (more correct) rather than getting a silently-recreated wrong-ID descriptor.

### 4.2 `ISO8601Millis.dateLenient` accepts fractional seconds

`string(from:)` writes timestamps **with** fractional seconds; `dateLenient(from:)` uses a formatter **without** them, so it returns `nil` on the app's own output (its doc-comment already *claims* it accepts both). Fix:

```swift
public static func dateLenient(from string: String) -> Date? {
    formatter.date(from: string) ?? lenient.date(from: string)   // fractional first, then non-fractional
}
```

Strictly more lenient — can't break existing parses. Removes the `CatalogSnapshot.verifyAdoption` `takenAtMs == 0` fallback for app-written mtimes.

## 5. Group D — Small Browse/Send adds

### 5.1 Video-only filter

A toggle in the timeline toolbar (and folders) to show only videos. `AppState` gains a `videoOnly: Bool` (persisted); the Core timeline query (`timelineSections`/`timelineItems`) gains an optional `videoOnly` predicate (`AND kind = 'video'`); the toolbar gets a toggle that flips it and calls `refreshQueries()`. Complements the existing `MediaKind` split.

### 5.2 Multi-device send picker

`connectedSendTarget()` returns the *first* receiver; when several are connected the user can't choose. Add `connectedSendTargets() -> [ConnectedDevice]` (all with a `sendDestination`). At send time, if there's more than one, present a small chooser (menu/confirmation dialog by friendly name, default = current auto-pick) before `SendSheet`; `SendSheet` already takes an explicit `device`, so `SendEngine`/destinations are untouched.

1. **Core (TDD)** — `Vault.open(at:)` + refactor `openOrCreate` to reuse it; `ISO8601Millis.dateLenient` fractional fix; the `videoOnly` query predicate. (Three small, independent Core changes with tests.)

5. **Docs** — master spec §10 (mark Phase 3.5 done) + changelog; next = consensus-repair slice, then Phase 4.

The `AppState.openVault(for:)` → `Vault.open` switch lands in **Task 1** (it's the one-line adoption of the new accessor; Task 1 verifies the whole package builds after the Core change). The tile work (Tasks 2–3) is the bulk; everything else is small.
