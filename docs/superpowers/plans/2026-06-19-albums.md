# Albums Implementation Plan

> **For agentic workers:** Implement task-by-task; build + test + commit each task. Steps use `- [ ]`.

**Goal:** Manual, ordered, drive-synced virtual photo collections stored sovereignly (one JSON per album in `.openphoto/albums/`), with a rebuildable catalog mirror and SwiftUI sidebar + grid.

**Architecture:** Core owns `AlbumRecord` (the JSON model), `AlbumStore` (sovereign file CRUD), and a catalog mirror (`albums`/`album_members`) rebuilt from the files. AppState exposes summaries + use-cases. UI adds an Albums sidebar section, a reorderable grid, and "Add to Album" menus. Members are content hashes (locked-gate-aware, drive-portable). Sync copies the album files to drives.

**Tech stack:** Swift, SwiftUI, GRDB, AtomicFile, MediaTile, SelectionActionBar. Spec: `docs/superpowers/specs/2026-06-19-albums-design.md`.

### Task 2: Catalog mirror + queries

**Files:**

- Modify: `Sources/OpenPhotoCore/Catalog/Catalog.swift` (add `v18` migration; bump `schemaVersion`)

- [ ] **Step 1 — Migration v18:** `albums(id TEXT PK, name TEXT NOT NULL, coverHash TEXT, createdAtMs INT NOT NULL, modifiedAtMs INT NOT NULL)`; `album_members(albumID TEXT NOT NULL, hash TEXT NOT NULL, position INT NOT NULL, PRIMARY KEY(albumID,hash))` with `hash` indexed. Bump `Catalog.schemaVersion` to next value; note in comment.
- [ ] **Step 2 — `AlbumSummary`:** `public struct AlbumSummary: Sendable, Identifiable, Equatable { id: String; name: String; coverHash: String?; count: Int }`.
- [ ] **Step 3 — Catalog+Albums methods:**
  - `replaceAlbums(_ albums:[AlbumRecord])` — txn: delete all rows in both tables, insert each album + members (position = index).
  - `upsertAlbum(_ album:AlbumRecord)` — txn: upsert album row, delete its members, re-insert in order.
  - `deleteAlbumMirror(id:)` — delete album + its members.
  - `itemsInAlbum(id:) -> [TimelineItem]` — `SELECT base.* FROM (instanceSQL) base JOIN album_members m ON m.hash = base.hash WHERE m.albumID = ? <lockedFilter> ORDER BY m.position` (dedup base to one row per hash first if instanceSQL yields multiple; use the same browse projection `item(hash:)` uses). Honor `lockedVisibilityClause`/`lockedFilter`.
  - `albumSummaries() -> [AlbumSummary]` — name + coverHash + count(distinct visible member hash, locked-gated).
  - `albumIDsContaining(hash:) -> Set<String>`.
- [ ] **Step 4 — Tests** (build a `Catalog` like `LockedFoldersTests` does): `replaceAlbums` then `itemsInAlbum` returns members in stored order; a member in a locked folder is absent when `revealLocked=false`, present when `true`; `upsertAlbum` reorders positions; dedup (same hash twice → one row); `albumIDsContaining` correct; `albumSummaries` count honors lock gate.
